# PERTURB: surrogate-only perturbation feature analysis

This folder contains the perturbation-study runner plus the single retained summary figure used for the perturbation analysis.

## Quick start

Run from the repo root using the existing venv:

```bash
./venv/bin/python -m PERTURB.run_study --n-bases 100 --out-csv PERTURB/perturb_long.csv
```

Defaults: `composition` sweeps `fPEO_w` from `0` to `1` in `0.1` increments.

To regenerate the retained figure:

```bash
./venv/bin/python -m PERTURB.make_mean_width_centered_bars
```

## What it generates

For each sampled base architecture, the runner writes a “baseline” row plus rows for each perturbation axis:

- `composition`: greedy chemistry flips `Sℓ↔Eℓ` at fixed lengths
- `sequence`: `shuffle` permutations over nonzero labels (zeros fixed; skipped when <2 nonzero sites)
- `grafting`: remove grafts by setting `Eℓ/Sℓ→E0/S0` with `periodic` removal
- `peo_scale`: scale PEO (`Eℓ`) lengths by a multiplier with clipping tracked as `pct_clipped` (keeps originally-grafted `E` sites grafted)
- `ps_scale`: scale PS (`Sℓ`) lengths by a multiplier with clipping tracked as `pct_clipped` (keeps originally-grafted `S` sites grafted)
- `dispersity`: redistribute lengths within chemistry at fixed per-chemistry total mass via an `alpha` mix

The long-form CSV includes:

- architecture metrics (`B`, `fPEO_w`, `sigma`, `ratio_ES`, `gini`, `cv`, `has_PEO`)
- totals/ordering/asymmetry metrics (`L_tot`, `L_E`, `L_S`, `n_trans`, `Eload_tot`, `Eload_E`, `Eload_S`, `cent_sep`)
- surrogate predictions (`A_pred`, `Rg_pred`, `RDF_pred`)
- deltas relative to the per-base baseline (`dA`, `dRg`, `dRDF`)
- an OOD score/flag computed from LR engineered descriptors fit on `data/ES.csv` (`ood_score`, `ood_percentile`, `ood_flag`)

## Retained figure

- `PERTURB/figs_mean_width/mean_width_centered_bars__overlay-boxplot_caps.png`
- Generated by `PERTURB/make_mean_width_centered_bars.py`
- Shows the mean signed max-min effect for each perturbation knob with boxplot-caps overlays for the per-base width distribution

## Notes

- The GIN surrogate in `PERTURB` loads the cross-validation model weights from `RESULTS_RANDOM_ONLY/cv/GIN/models`.
- `B` (blockiness) is computed over nonzero sites only and is `NaN` when there are fewer than 2 grafted sites.
- `gini` and `cv` are computed over nonzero sidechain lengths only (ungrafted sites are ignored).
- OOD scoring is based on LR engineered features + scaling + (optional) PCA whitening; it is intended as a guardrail, not a hard validity check.
- RDF interpretation: the repository datasets encode “no PEO sidechains” as `has_PEO=0` (all `Eℓ` are `E0`), where ground-truth RDF Peak is typically `0`; you can filter RDF plots to `has_PEO==1`.
